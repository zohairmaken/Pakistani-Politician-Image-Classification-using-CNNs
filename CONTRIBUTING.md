# Contributing to Pakistani Politician Image Classification

Thank you for your interest in contributing! We follow a professional GitHub workflow to maintain code quality and project integrity.

## 🛠️ Development Workflow

1. **Fork the Repository**: Create a personal copy of the project.
2. **Create a Feature Branch**: Use descriptive names like `feature/new-model` or `bugfix/data-loader`.
3. **Commit Changes**: Follow conventional commit messages (e.g., `feat:`, `fix:`, `docs:`, `refactor:`).
4. **Pull Requests**: Submit a PR to the `development` branch. Provide a clear description of your changes.
5. **Code Review**: All PRs require review from at least one maintainer.

## 📏 Code Standards
- Follow PEP 8 guidelines.
- Include docstrings for all new functions and classes.
- Ensure type hinting is used where applicable.
- Add unit tests for core logic.

## 🧪 Testing
Run tests using `pytest`:
```bash
pytest tests/
```
